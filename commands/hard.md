這是一個困難任務，啟動高規格流程。任務描述：$ARGUMENTS

先讀 ~/.claude/rules/judgment.md 與 ~/.claude/rules/delegation.md，然後嚴格照以下順序執行，不可跳步：

1. **先不寫任何 code。** 用自己的話複述需求，列出（a）你確定的部分（b）不確定、需要假設的部分。不確定的部分若會改變做法，現在一次問完（最多 3 題，每題附你的預設答案）。
2. **出 2–3 個可行方案**，每個附：做法一句話、風險、大約工作量、你推薦哪個與為什麼。等使用者選定才進下一步（使用者若已在任務描述中指定做法，跳過此步）。
3. **先寫驗收條件再動工**：列出「怎樣算完成」的可檢查清單（行為改變、測試、使用者要看哪裡）。
4. **拆解與派工**：照 delegation.md 執行——大量讀取派 Explore、實作可派 sonnet、你只負責指揮與整合。難的子問題用 opus。
5. **完成後不可自驗**：照 delegation.md 第 6 節派 fresh-context agent 對照第 3 步的驗收條件逐條檢查，把結果原文附在回報裡。
6. **回報格式**：結果一句話 → 驗收清單逐條打勾/打叉 → 使用者要看哪裡、預期看到什麼 → 殘留風險。
