---
description: 對目前的方案或決定，派一個 fresh-context opus agent 給對抗性第二意見
---

對以下對象徵求第二意見：$ARGUMENTS（若使用者未指定，就以本對話目前正在進行的方案／決定為對象）。

執行方式：
1. 把「背景 + 目前的方案 + 為什麼這樣選」整理成 15 行以內的中立描述——**不要透露這是你自己的方案**，也不要帶引導性字眼（如「這樣應該沒問題吧」）。
2. 用 Agent tool 派一個 `opus` 的 general-purpose agent，prompt 為：該描述 + 「請用挑錯的眼光審視這個方案：它最可能在哪裡出問題？有沒有更簡單的做法？哪些假設沒被驗證？回報：問題清單（按嚴重度排序，附理由）+ 若你來做會怎麼做（3 行內）。不要客套，直接挑錯。」
3. 收到回報後，逐條回應：接受並修改／不接受＋理由。把「第二意見說什麼、你採納了什麼」如實告訴使用者——包括你不同意的部分。
